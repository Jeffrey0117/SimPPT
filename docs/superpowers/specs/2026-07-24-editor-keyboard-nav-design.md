# 編輯器鍵盤翻頁 + 頁碼文字樣式

2026-07-24 定案。編輯器右側預覽/縮圖列目前只能點縮圖切換頁面，加鍵盤快捷鍵；順便修頁碼文字太灰看不清楚。

## 決策脈絡

- 單純方向鍵（↑↓←→）已經是 textarea 移動文字光標用的，不能直接拿來翻頁，會打斷打字。改用 `Ctrl+方向鍵`（上下左右都通，不分語意方向，都對應上一頁/下一頁的其中一個）。

## 變更

1. **鍵盤翻頁**：編輯器內按 `Ctrl+↑`／`Ctrl+←` = 上一頁；`Ctrl+↓`／`Ctrl+→` = 下一頁。呼叫既有的 `jumpEditorToSlide(idx)`（跟點縮圖同一套邏輯），`idx` 從目前的 `previewIdx` 加減 1，並夾在 `[0, deck.slides.length - 1]` 範圍內；到頭/尾時該方向不動作（不循環）。
2. **頁碼文字樣式**：`#ed-pageinfo` 從 `font-size: 12.5px; color: #77777f` 改成 `font-size: 15px; color: #f5f5f5`（跟編輯器其他白字一致）。

## 測試範圍

純 UI 互動 + CSS 調整，不寫自動測試；語法檢查（`new Function`）+ 手動瀏覽器驗證：在編輯器按 `Ctrl+方向鍵` 確認縮圖列/預覽正確換頁、到頭尾不出錯；頁碼文字肉眼確認變白變大。

## 不做（YAGNI）

不做循環翻頁（到最後一頁再按下一頁不會繞回第一頁）、不改變單純方向鍵的既有文字游標行為。
